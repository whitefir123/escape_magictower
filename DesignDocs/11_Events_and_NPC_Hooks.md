# NPC 服务者与事件驱动模型 (NPC Entities & Event Hooks)

## 一、 系统钩子与生命周期广播 (Global Event Broadcaster)
这个游戏拥有大量的“满足条件触发”和“在死亡边缘拉你一把”的装备词缀与符文。如果不用发布者/订阅者（Pub/Sub）模式，代码会变成一坨难以解开的乱麻。

### 1.1 核心战场生命周期钩子
整个游戏必须围绕一个中央的 `EventManager` 散发以下几个核心广播钩子。任何想要生效的符文、装备、天赋，必须去**订阅这些钩子，然后在对应时刻插队执行自己的运算逻辑后再返回主线**。

- **On_Battle_Start (战斗开局检测)**：
  - 触发场合：刚切入战斗状态的第 0 回合前。
  - 适用方：比如“盗贼的暗影斗篷：开局必定获得100%闪避持续1回合”。此逻辑监听该钩子，触发后给自己上一层增益。
- **On_Entity_Take_Damage_Before (承伤结算前)**：
  - 触发场合：已经算完所有公式得出理论伤害，正准备真扣血的刹那间。
  - 适用方：比如“圣骑士免死金牌：受到致命伤时，伤害归零并免疫后续一切一回合”。此逻辑检查该次伤害是否 >= 当前血线，若是，则劫持并修改伤害数字。
- **On_Entity_Kill (成功击杀判定)**：
  - 触发场合：某个怪物流血流死、或被一刀砍碎化为灰烬后。
  - 适用方：比如“嗜血符文：每击杀一次怪永久加血上限”。
- **On_Floor_Enter (进门清算)**：
  - 触发场合：玩家踩扁上一层Boss，爬进下一层休息室的那一刻。
  - 适用方：某些诅咒符文的发作期或者铁匠铺重置自身折扣。

## 二、 城镇化身机制：特殊 NPC 构建

### 2.1 铁匠：奥恩 (The Blacksmith)
这不仅仅是一个提供按键强化的机器，除了在 `04` 号文档介绍的机能外，铁匠拥有“隐藏好感度积累系统”。
- 每当玩家大量砸入金币进行强化、甚至在这里不幸强化失败时，系统后台都会累计他的“打铁数”。
- **回报**：铁匠会渐渐赋予玩家“贵宾折扣”，或者小概率在你的破铜烂铁重铸时免费塞入一枚高阶前缀。

### 2.2 黑市导盲者：行商 (The Wandering Merchant)
行商在地下城深层属于偶尔出现的绿洲。
- **商品分级陈列法则**：他的商品不是一成不变的，是严格遵循“3个消耗品栏位，2个昂贵盲盒或高阶图纸栏位，1个诅咒珍宝栏位”的结构定桩投放。并且，这些栏位所摆放的物品必定参考玩家当前深入的楼层来匹配其 iPwr（物品强度）基数。

### 2.3 命运赌局：贪婪之影 (The Gambler)
- 游戏机逻辑机制：每一次抽奖行为不仅消耗剧增的金币，实质上是在隐蔽地执行一次“极低 Magic Find 干预下的最高规格掉落生成计算（参阅08_Loot生成）”。但由于它的底盘起点极高，使得这是玩家唯一可能在第 5 层就拿到本应该在第 20 层才会产出的逆天词缀的途径。

## 三、 特殊事件点：祭坛系统 (Shrines)
这些点位不需要玩家支付金币，它们的存在是为了扰乱枯燥的公式推演。
- **祈福祭坛**：点亮后给玩家带来一种能够持续到本层结束的“变态Buff”（例如：移动不受任何陷阱判定干扰）。
- **献祭祭坛**：主动交互后，永久性地从玩家身上剥离一部分最大生命值上限（割肉），换取在此刻立即提供的一件指定部位的高阶顶级防具（如红装或彩装）——极其残酷的资源转嫁考验。
