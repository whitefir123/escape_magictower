## 【阶段一：你的架构与编码铁律】

1. **彻底解耦**：禁止 `GameObject.Find`，禁止单例模式满天飞。使用依赖注入或服务定位器（ServiceLocator）。
2. **事件总线驱动 (Event-Driven)**：EventBus / Pub-Sub。"玩家受击"、"掉落金币"、"触发融化"等一切交互必须走中央 EventManager 广播。绝不能直接从怪物脚本里去扣玩家血量！
3. **数据驱动配置 (Data-Driven)**：使用 ScriptableObjects (SO) 或 JSON 来承载所有配置表。严禁将数值写死在 `.cs` 文件里。
4. **状态机至上**：所有 Buff/Debuff 必须经由全局 StatusEffectManager 统一接管生命周期。不可叠加型（冰封、眩晕）刷新时间；无限叠层型（中毒、灼烧、流血、破甲）+1层重置计时。详见 `DesignDocs/09_DataFlow_and_Status.md`。
5. **属性管线全量重算**：面板不允许使用增量逻辑（`+= new - old`）！任何层级变动后（换装、选符文、Buff过期），必须在六层管线上走一遍完整重算。详见 `DesignDocs/09_DataFlow_and_Status.md`。
6. **指令缓存队列 (Command Buffer)**：迭代中不可增删实体。所有死亡/生成/移除操作写入队列，`LateUpdate` 统一处理。详见 `DesignDocs/13_Architecture_and_Operations_SLA.md`。
7. **被动触发熔断器**：事件嵌套 Generation > 3 层立即熔断，防止 A→B→C→A 死循环。
