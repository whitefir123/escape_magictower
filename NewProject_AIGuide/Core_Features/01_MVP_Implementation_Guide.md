# 逃离魔塔 - 重生计划 (Core MVP 核心框架篇 v2.0)

**【写给主程 AI 的核心准则】**
第一阶段的唯一目标：**跑通游戏最核心的战斗循环 (Core Loop)**。
请完全无视装备掉落、商店、复杂的流派构建等冗余外围系统。
所有具体的数值与机制，请严格引用 `DesignDocs/` 与 `GameData_Blueprints/` 目录下的设计文件。

---

## 核心阶段唯一目标：MVP (Minimum Viable Product) 跑通

### 1. 玩家进入与英雄控制
- 玩家能够成功进入游戏，利用 **WASD 键盘** 或 **鼠标右键点击地板** 控制英雄在 2D 场景中移动。
- 按下 **F 键** 或 **鼠标左键** 可与地图中的可交互实体互动。
- **参考数据**：英雄的基础移动速度、血量、攻击逻辑请严格参考 `DesignDocs/02_Entities_and_Stats.md` 中的属性定义。职业技能树（MVP阶段选用【流浪剑客】作为首发）请参考 `GameData_Blueprints/05_Hero_Classes_And_Skills.md`。经验条初始 EXP 0/100，递增规则同文件内定义。
- **操作映射**：所有非移动按键必须支持在设置面板中自定义。详见 `DesignDocs/07_UI_and_UX.md` 第三节。

### 2. 战场加载与无限扩展架构预留
- **地图呈现**：当前阶段**仅实装第一层【暗黑地牢】的生态地图与刷怪池**。
- **【架构红线警告】**：虽然暂只做一张地图，但你在构建 `MapManager` 和生态生成器时，必须**预留支持共计 9 大生态层的底层接口**。地图结构必须支持无缝的背景池切换和 `BiomeConfig_SO` 式的群系读取。
- **参考数据**：`GameData_Blueprints/03_World_Themes.md` 为九大生态主题，`DesignDocs/06_Map_and_Modes.md` 为地图节点与事件设计。

### 3. 怪物的生成与类魔塔战斗交互
- **索敌与碰撞**：实现纯正的类魔塔不对称碰撞机制——
  - **近战**：玩家向怪物格子发起碰撞视为一次"主动攻击"，怪物**立即无条件被动反击一次**（经典换血）。
  - **远程**：鼠标右键点选远处敌方单位发射投射物。怪物进入追击后，远程角色自动锁定最近追击目标自动射击。
  - **怪物主动追击**：被攻击后怪物进入仇恨链，主动冲锋玩家。**但玩家被打时绝不自动反击。**
- **伤害结算**：实现完整的"威力→暴击→穿甲综合结算(含跨界穿透)→递减防御→最终修正"链路。详见 `DesignDocs/03_Combat_System.md`。
- **软狂暴**：若战斗过长则怪物叠加疲劳层数逼迫玩家进攻。
- **参考数据**：怪物面板与同心圆法则见 `GameData_Blueprints/04_01_Monster_Spawn_Logic.md`，第一层怪物百科见 `GameData_Blueprints/04_1_Floor1_Dungeon.md`。

### 4. 独特的战利品循环（经验与属性符文）
- **击杀掉落**：玩家击杀怪物时，有 **40%** 的概率触发【属性强化符文】三选一。
  - 触发后，**游戏必须强制暂停**，弹出 UI 界面供玩家选择纯面板属性增强。
- **升级奖励**：玩家经验槽满后升级，强制弹出**【机制符文三选一】**系统。
  - 保底机制：连续15次（升5级）未见史诗及以上，强制干涉出史诗档位。
- **参考数据**：符文奖池与权重严格引用 `DesignDocs/05_Runes_and_MetaProgression.md` 与 `GameData_Blueprints/08_Destiny_Rune_System.md`。

### 5. 存档与状态管线
- **手动存档 + 自动存档**：玩家可在安全区手动存盘，系统在通关关卡时自动存盘。**默许 SL 大法，但随机种子不可逆。**
- **属性管线**：任何装备/符文/Buff变动后，必须在六层属性管线上全量重算。绝不允许增量逻辑。
- **参考数据**：`DesignDocs/09_DataFlow_and_Status.md` 与 `DesignDocs/10_Audio_and_Persistence_Rules.md`。

### 6. Boss 战与跑通胜利
- 第一层关底设置一位 Boss，具备独有的 AI 行为模式（带有特殊机制而非纯属性碾压）。
- 玩家击败 Boss 后，弹出【胜利 UI (Victory)】，至此 MVP 闭环彻底跑通。

---

## 架构底线强制约束（违反任何一条将导致代码审计不通过）

1. **事件总线**：所有跨系统通信走 EventBus，禁止脚本之间直接引用。
2. **数据驱动**：全部数值从 ScriptableObject / JSON 加载，`.cs` 里零 hardcode。
3. **指令缓存**：战斗中实体的死亡/新增操作统一写入 Command Buffer Queue，`LateUpdate` 批处理。
4. **被动触发熔断**：事件嵌套超过 Generation 3 层立即强制熔断。
5. **钳制边界**：所有属性必须经过 `Clamp`。减伤率硬上限 `99%`，跨界转化率各有独立硬上限。
